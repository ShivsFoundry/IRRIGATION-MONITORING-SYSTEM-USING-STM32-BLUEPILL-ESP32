# Contributing

Thank you for your interest in contributing to the **IRRIGATION MONITORING SYSTEM USING STM32 BLUEPILL & ESP32** project.

Contributions from the community are highly appreciated and help improve the quality, reliability, and usability of this project.

---

## Ways to Contribute

You can contribute by:

- Fixing bugs
- Improving documentation
- Optimizing the firmware
- Enhancing code readability
- Supporting additional STM32 or ESP32 boards
- Adding new sensors
- Improving IoT functionality
- Adding new cloud platforms
- Reporting issues
- Suggesting new features

---

## Before You Start

Please ensure that you:

- Read the project documentation.
- Search existing issues before creating a new one.
- Open an issue before making major changes.
- Keep pull requests focused on a single improvement.

---

## Development Guidelines

Please follow these guidelines while contributing.

### Code Style

- Write clean and readable code.
- Use meaningful variable names.
- Maintain the existing project structure.
- Comment important sections of the code.
- Avoid unnecessary code changes.
- Preserve existing functionality unless intentionally improving it.

---

## Commit Messages

Use clear and descriptive commit messages.

Example:

```text
Fix UART communication issue

Improve README documentation

Add support for DHT22 sensor

Optimize ThingSpeak upload routine
```

---

## Pull Request Process

1. Fork this repository.
2. Create a new branch.

```bash
git checkout -b feature/YourFeature
```

3. Make your changes.

4. Commit your work.

```bash
git commit -m "Describe your changes"
```

5. Push to your fork.

```bash
git push origin feature/YourFeature
```

6. Open a Pull Request.

---

## Reporting Bugs

When reporting bugs, please include:

- Board model
- Arduino IDE version
- Operating system
- Library versions
- Serial Monitor output
- Steps to reproduce
- Expected behavior
- Actual behavior

---

## Feature Requests

Feature requests are welcome.

Please include:

- Description of the feature
- Use case
- Expected benefit
- Possible implementation approach

---

## Documentation Improvements

Documentation contributions are always appreciated.

Examples include:

- README improvements
- Better wiring diagrams
- Additional tutorials
- Code comments
- Troubleshooting guides

---

## Code of Conduct

Please be respectful and constructive when interacting with the community.
